# 전기 교육 및 기술 서비스 홈페이지 구축 계획

## 1. 프로젝트 목표
전기 기술 서비스 및 자격증 교육을 제공하는 회사 홈페이지 구축.
교육 신청, 솔루션 문의, 데이터 관리까지 가능한 구조로 설계.

### 핵심 목표
- 회사 신뢰성 확보 (전문분야 소개, 기술력 강조)
- 교육 신청 흐름 최적화 (과정 → 신청 → 입금 안내)
- 솔루션 문의 리드 수집
- 신청 데이터를 DB에 누적하여 향후 회원 시스템 확장 가능

---

## 2. 기술 스택 (확정)

### 프론트엔드 및 배포
- Next.js (App Router)
- TypeScript
- Vercel 배포

장점:
- 자동 HTTPS
- 빠른 배포
- Preview 환경 제공

### 신청 폼
- Tally 사용
- Webhook으로 서버에 데이터 전달

### 백엔드 및 데이터베이스
- Supabase (PostgreSQL 기반)

사용 목적:
- 교육 과정 데이터 저장
- 신청 데이터 저장
- 솔루션 문의 관리

### Webhook 처리 방식
- Next.js API Route (/api/tally-webhook)
- Tally → API Route → Supabase 저장

---

## 3. 구축 단계 (Phase 전략)

### Phase 1 (MVP — 회원가입 없음)
- 홈페이지 제작
- 교육 과정 리스트
- 교육 일정 표시
- Tally 신청폼 연결
- 통장입금 안내 페이지
- Webhook으로 DB 자동 저장

운영 방식:
- Supabase에서 신청 데이터 조회
- 상태 관리 (입금대기 / 입금확인 / 취소)

### Phase 2 (회원 시스템 추가)
- Supabase Auth 로그인
- 회원가입 기능
- 마이페이지
  - 내 신청 내역
  - 결제 상태 확인
  - 교육 자료 다운로드

### Phase 3 (자동화 확장)
- 입금 확인 시 자동 안내메일
- 입금 미확인 알림
- 문자/알림톡 연동

## 6. Webhook 처리 흐름

1. 사용자가 Tally 폼 제출
2. Tally가 webhook URL 호출
3. Next.js API route가 데이터 수신
4. Supabase DB 저장
5. 완료 응답

보안:
- webhook secret 사용
- 요청 검증
